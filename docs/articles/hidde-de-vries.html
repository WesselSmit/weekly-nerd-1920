Modernism and simplifying everything

Hidde talked about how in the current web standard function is above form. A trend that started with modernism.

On the web the possibilities are limitless and it's a shame that nowadays most developers like to play it safe. Websites nowadays are all very similar and as developers we should aim more for the wow-effect on the web!

Internet follows in the footsteps of modernism; modernism simplifies everything constantly. Take company logo's for example, these have increasingly changed from illustrations to calligraphic letters, these have been simplified over and over. Today, most logos look like the KLM logo below; safe & boring.

The current web

More and more is possible with modern technologies and newer features, which is increasingly visible on the web as companies play it safe. For example, they use the older but better supported features to create their websites because this works for most people in the browser. Hidde emphasizes that this is the wrong method; use the newer technologies and use fallbacks if necessary.

Do not compromise the design, but offer alternatives where necessary.

The web is accessible, flexible & interactive by default

The web standards strive for a web that is usable for everyone, so Hidde's biggest takeaway to us was: stay as close to the standards as possible, try to do as much vanilla as possible. When you write semantic HTML, your application already has a very strong foundation for accessibility; your website is tablable, screen readers can interpret the content well & basic interactions are possible!
